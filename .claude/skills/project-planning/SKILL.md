# 바이브코딩 프로젝트 기획 워크플로우

프로젝트를 체계적으로 기획하고 개발하기 위한 단계별 가이드입니다.

## 핵심 원칙

> **"개발 전에 기획, 기획 전에 검증"**

코드를 작성하기 전에 무엇을 만들지 명확히 정의하고, UI/UX를 시각화하여 방향성을 확인합니다.

---

## 표준 워크플로우 (4단계)

```
1단계: 프로젝트 초기 설정
   ↓
2단계: PRODUCT_SPEC 작성 (MVP 확인)
   ↓
3단계: 목업 제작 (UI/UX 시각화)
   ↓
4단계: 개발 진행
```

---

## 2단계: PRODUCT_SPEC 작성

### 목적
- 프로덕트 아이디어를 구체화
- 요구사항 명확히 정의
- 개발 방향 합의

### 진행 방법: 프로덕트 기획 인터뷰 (9단계)

#### Step 0: 초기 아이디어 듣기
**질문:**
1. 어떤 프로덕트를 만들고 싶으신가요?
2. 이 아이디어를 어떻게 떠올리게 되셨나요?
3. 간단히 한 문장으로 설명한다면?

**산출물:** 프로덕트 아이디어 한 문장 요약

---

#### Step 1: 배경 이해하기
**질문:**
1. 현재 상황은 어떤가요? (시장, 조직, 사용자 측면)
2. 이 프로덕트가 필요한 이유는 무엇인가요?
3. 비슷한 솔루션이나 경쟁 제품이 있나요? 있다면 어떤 점이 다른가요?

**산출물:** 배경 & 현황 요약

---

#### Step 2: 문제 정의하기
**질문:**
1. 해결하려는 핵심 문제는 무엇인가요?
2. 이 문제로 인해 누가 어떤 불편을 겪고 있나요?
3. 이 문제를 해결하지 않으면 어떤 일이 발생하나요?
4. 문제를 한 문장으로 정의한다면?

**산출물:** 핵심 문제 정의 (한 문장)

---

#### Step 3: 타겟 사용자 파악하기
**질문:**
1. 이 프로덕트의 주요 사용자는 누구인가요?
2. 사용자의 직급, 역할, 업무 환경은 어떤가요?
3. 사용자가 현재 이 문제를 어떻게 해결하고 있나요?
4. 사용자가 이 프로덕트를 얼마나 절실하게 필요로 하나요? (1-10점)

**산출물:** 페르소나 정의
- 이름, 나이, 직급
- 업무 및 과제
- 현재 해결 방법
- 절실함 점수

---

#### Step 4: 목적과 목표 설정하기
**질문:**
1. 이 프로덕트를 통해 달성하고 싶은 목표는 무엇인가요? (3개월, 1년)
2. 성공을 어떻게 측정할 건가요? (핵심 지표)
3. 1년 후 이 프로덕트가 어떤 모습이기를 바라나요?

**산출물:**
- 비전 (1년)
- 미션 (한 문장)
- 핵심 KPI (2-3개)

---

#### Step 5: 솔루션 구체화하기
**질문:**
1. 이 프로덕트가 문제를 어떻게 해결하나요?
2. 기존 방법과 어떤 점이 다른가요? (차별점 3가지)
3. 사용자가 이 프로덕트를 사용하는 과정을 단계별로 설명한다면? (사용자 여정)

**산출물:**
- 솔루션 한 줄 설명
- 핵심 차별점 3가지
- 사용자 여정 (7단계 이하)

---

#### Step 6: 핵심 기능 정의하기
**질문:**
1. MVP(최소 기능 제품)에 반드시 포함되어야 할 기능은 무엇인가요? (3-5개)
2. 나중에 추가할 수 있는 기능은 무엇인가요?
3. 각 기능의 우선순위는 어떻게 되나요?

**산출물:**
- MVP 필수 기능 목록 (3-5개)
- 향후 추가 기능 목록

---

#### Step 7: 비즈니스 모델 설계하기
**질문:**
1. 이 프로덕트의 비용 구조는 어떻게 되나요? (API 비용, 인프라 등)
2. 투자 대비 기대 효과(ROI)는 어떻게 예상하나요?
3. 지속 가능성을 위한 계획은 무엇인가요?

**산출물:**
- 비용 구조
- 기대 효과 (ROI)

---

#### Step 8: 실행 계획 수립하기
**질문:**
1. 개발 일정은 어떻게 계획하시나요? (주차별)
2. 예상되는 리스크는 무엇인가요?
3. 각 리스크에 대한 대응 전략은 무엇인가요?

**산출물:**
- 주차별 개발 계획
- 리스크 & 대응 전략 표

---

## 3단계: 목업 제작

### 목적
- UI/UX 사전 시각화
- 사용자 플로우 검증
- 개발 전 디자인 확정
- 개발 시 명확한 레퍼런스 제공

### 목업 제작 방법

#### 옵션 1: HTML/CSS 프로토타입 (추천) ⭐

**장점:**
- 실제 개발 코드의 기반
- 클릭 가능한 인터랙티브 프로토타입
- Tailwind CSS로 빠른 스타일링
- 반응형 테스트 가능

**방법:**
```bash
# 프로젝트 내에서 정적 UI만 먼저 구현
cd projects/[project-name]

# 각 페이지 UI만 만들기 (기능 없음)
# app/page.tsx - 정적 UI
# app/create/page.tsx - 정적 UI
```

#### 옵션 2: 와이어프레임 문서

**방법:**
`MOCKUP.md` 파일 생성

```markdown
# MOCKUP.md

## 1. 홈 페이지
┌─────────────────────────────┐
│  프로젝트 타이틀            │
├─────────────────────────────┤
│  [+ 새로 만들기]            │
├─────────────────────────────┤
│  📂 내 작업물               │
│  ├ 항목1                   │
│  ├ 항목2                   │
└─────────────────────────────┘

## 2. 생성 페이지
[와이어프레임]
```

#### 옵션 3: 디자인 툴

**도구:** Figma, Excalidraw, Sketch

**장점:**
- 시각적으로 정교함
- 디자이너와 협업 용이

**단점:**
- 시간이 더 걸림
- 코드로 변환 필요

---

### 체크리스트

**PRODUCT_SPEC 작성**
- [ ] 프로덕트 기획 인터뷰 9단계 완료
- [ ] `PRODUCT_SPEC.md` 파일 생성 (프로젝트 폴더 내)
- [ ] 핵심 내용 검증
  - 문제 정의 명확한가?
  - 타겟 사용자 구체적인가?
  - 사용자 여정 7단계 이하로 단순한가?
  - MVP 기능이 명확한가?
  - 성공 기준이 측정 가능한가?

**목업 제작**
- [ ] 목업 방법 선택 (HTML/와이어프레임/디자인툴)
- [ ] 모든 화면 목업 제작 (사용자 여정 각 단계)
- [ ] 스타일 가이드 정의
  - 색상 팔레트
  - 폰트 (제목, 본문)
  - 간격, 버튼, 카드 스타일
- [ ] 사용자 플로우 검증
- [ ] 목업 리뷰 및 승인

## 프로젝트 크기별 가이드

| 프로젝트 크기 | 워크플로우 사용 | 예시 | 기간 |
|--------------|---------------|------|------|
| **대형** | 1→2→3→4 전체 | 새로운 SaaS, 복잡한 AI 기능 | 4-8주 |
| **중형** | 1→2(간략)→3→4 | 주요 기능 추가, 3-5 페이지 | 2-4주 |
| **소형** | 바로 4 (개발) | 버그 수정, UI 개선 | 1-5일 |
